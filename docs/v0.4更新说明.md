# 灵犀 API Client v0.4.0 更新说明

> 发布日期：2025-06-04

## 概述

v0.4.0 是一个功能完善版本，新增了 HTTP 代理抓包、Toast 通知系统、请求重放增强、E2E 测试框架、Cookie Jar 完整功能，以及多项 UI/UX 改进。

## 新增功能

### 1. HTTP 代理抓包

- **基础 HTTP 代理**：支持 GET/POST/PUT/PATCH/DELETE 等方法的 HTTP 代理抓包
- **代理服务管理**：可在"请求捕获 / HTTP 代理"页面启动/停止代理
- **默认代理地址**：`http://127.0.0.1:8899`，可在设置中修改端口
- **日志筛选**：支持按 method、status_code、host、keyword、HTTPS CONNECT 筛选
- **记录详情**：可查看请求头、请求体预览、响应头、响应体预览
- **HTTPS CONNECT**：记录目标域名和端口，**不解密 HTTPS 内容**（安全考量）
- **代理端口设置**：设置页面新增代理端口输入

### 2. Toast 通知系统

- 统一 Toast 组件，覆盖 success / error / info / warning 四种类型
- 中文文案，3 秒自动消失，可手动关闭
- 覆盖场景：
  - 设置保存成功/失败
  - Cookie Jar 操作结果
  - 代理启动/停止成功/失败
  - 复制成功
  - 请求重放成功/失败
  - 清空数据成功/失败
- 使用方式：`import { toast } from '@/shared/ui/Toast'` → `toast.success('消息')`

### 3. Cookie Jar

- **自动保存**：收到 Set-Cookie 响应头时自动保存到 SQLite
- **自动携带**：后续同域名、同 path 匹配的请求自动携带 Cookie
- **匹配规则**：domain/path/expires/secure/http_only/same_site 完整匹配
- **过期处理**：过期 Cookie 自动过滤，不会发送
- **Secure Cookie**：仅通过 HTTPS 请求发送
- **手动优先**：手动设置的 Cookie Header 优先级高于 Cookie Jar 自动匹配
- **启用/禁用**：设置页面提供 Cookie Jar 开关
- **清空功能**：支持清空所有 Cookie（含确认弹窗）
- **Cookie 面板**：查看已保存 Cookie 的完整属性

### 4. 请求重放增强

- **从历史恢复**：将历史记录恢复为当前请求配置，可修改后发送
- **从捕获恢复**：将捕获记录恢复到工作台
- **立即重放**：弹窗确认后直接重新发送（敏感 Header 时追加提醒）
- **Toast 反馈**：重放成功/失败显示 Toast 通知
- 重放产生新的历史记录

### 5. 设置页面增强

- **Toast 通知**：所有设置变更操作均显示成功/失败 Toast
- **Cookie Jar 管理**：新增 Cookie Jar 启用/禁用开关、Cookie 数量显示、清空按钮
- **代理端口设置**：新增 HTTP 代理抓包端口配置
- **清空数据确认**：清空数据按钮增加确认弹窗和结果 Toast
- **UI 优化**：各设置区域统一排版，信息提示更清晰

### 6. Monaco Editor 增强

- **中文语言包**：加载 Monaco Editor 中文翻译，菜单和提示信息中文化
- **快捷键系统**：
  - `Ctrl/Cmd + Enter` — 发送请求
  - `Ctrl/Cmd + S` — 保存请求
  - `Alt/Option + Shift + F` — 格式化 JSON
- **快捷键提示栏**：编辑器下方显示当前可用的快捷键说明

### 7. E2E 测试

- **Playwright 测试**：覆盖核心用户流程的端到端测试
- **GitHub Actions**：支持手动触发的 E2E CI 工作流
- **本地运行**：`python3 scripts/e2e.py` 或 `npm run e2e`

### 8. 快捷键提示

- **请求编辑器提示栏**：URL 输入区下方显示快捷键说明
- KBD 样式显示快捷键键位
- 显示操作说明：发送、保存、格式化 JSON

| 方法 | 端点 | 说明 |

| POST | `/api/proxy/start` | 启动代理 |
| POST | `/api/proxy/stop` | 停止代理 |
| GET | `/api/proxy/logs` | 获取代理日志 |
| GET | `/api/proxy/logs/{id}` | 获取单条日志 |
| DELETE | `/api/proxy/logs` | 清空代理日志 |

## 已知限制

1. **HTTPS MITM 解密**：不支持，也不会实现
2. **HTTPS CONNECT 限制**：仅记录域名和端口，不解密内容
3. **E2E 测试**：需手动触发，非 CI 默认运行
4. **WebSocket 调试**：计划后续版本
5. **GraphQL 调试**：计划后续版本
6. **桌面端打包**：计划 v1.0

## 升级指南

1. 更新代码：`git pull origin main`
2. 后端依赖：`cd backend && .venv/bin/pip install -r requirements.txt`
3. 前端依赖：`cd frontend && npm install`
4. 构建前端（单服务模式）：`cd frontend && npm run build`
5. 启动：`cd backend && .venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 17321`

首次启动后端会自动创建新数据库表（`CookieJar`），无需手动迁移。
