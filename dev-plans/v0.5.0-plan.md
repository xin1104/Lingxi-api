## 一、技术决策

### 1.1 Tests 执行引擎：Python 后端执行

**决策**：Tests 脚本在 Python 后端执行（非前端 `new Function()`）。

**理由**：
1. 前端 `testRunner.ts`（已存在）仅在前端演示用，无沙箱隔离，无法访问环境变量和数据库
2. 后端执行可注入完整上下文（环境变量、全局变量、请求/响应对象）
3. 后端 `exec()` + 受限 `globals` 实现可控沙箱，安全且可扩展
4. 现有 `Pre Script` 也是前端执行（通过 store 桥接），v0.5.0 将其也迁移到后端统一执行

**脚本语法**：保持类 JavaScript DSL（与现有前端 testRunner 语法一致）：
```javascript
pm.test("状态码为 200", () => {
  pm.expect(pm.response.status).toBe(200)
})

pm.test("响应体包含 success", () => {
  pm.expect(pm.response.body).toContain("success")
})

pm.test("返回 JSON 格式", () => {
  pm.expect(pm.response.json()).toBeJson()
})

pm.test("响应时间小于 500ms", () => {
  pm.expect(pm.response.time).toBeLessThan(500)
})
```

Python 实现方式：使用 `exec()` 在受限的 `globals` 命名空间中执行脚本字符串。`pm` 对象是纯 Python 类实现的 DSL，语法与前端 testRunner 完全一致。

### 1.2 Pre Script 迁移到后端

现有 Pre Script 在前端 store 的 `sendCurrentRequest()` 中执行，修改 headers/params/body 后再发送到后端。v0.5.0 改为后端统一执行，将 Pre Script + Test Script 一起发给后端，后端执行 Pre → 发送请求 → 执行 Tests → 返回全部结果。

**API 变更**：`POST /api/send-request` 扩展请求体，新增 `pre_script` 和 `test_script` 字段。

### 1.3 HTML 预览安全方案

使用前端 iframe `sandbox` 属性渲染 HTML 响应：
- `sandbox="allow-same-origin"` — 允许访问同源资源
- CSP header: `Content-Security-Policy: default-src 'none'; style-src 'unsafe-inline'`
- 后端不做 HTML 处理，前端创建 `blob:` URL 后嵌入 iframe

### 1.4 二进制下载方案

后端在 `ApiResponseData` 中新增 `is_binary` 标志。前端检测二进制响应时渲染下载按钮，通过 `Blob` + `URL.createObjectURL()` 触发浏览器下载。

## 三、后端改动

### 3.1 新增：`backend/app/features/request_workbench/test_runner.py`

Tests 执行引擎，在 Python 后端实现类 JS DSL。

```python
class TestRunner:
    """在后端执行 Tests 脚本（DSL 类 JS 语法）"""

    def __init__(self, response: ApiResponseData, env_vars: dict):
        self.context = {
            "response": response,
            "env_vars": env_vars,
        }
        self.results: list[dict] = []

    def run(self, script: str) -> list[dict]:
        """执行测试脚本，返回测试结果列表"""
        if not script or not script.strip():
            return []

        # 构建 pm 对象
        pm = PmObject(self.context, self.results)

        try:
            exec(script, {"pm": pm, "__builtins__": {}})
        except Exception as e:
            self.results.append({
                "name": "脚本执行错误",
                "passed": False,
                "message": str(e),
                "expected": "",
                "actual": "",
            })

        return self.results
```

`PmObject` 提供 `pm.test()`, `pm.expect()`, `pm.response`, `pm.environment` 等 API，语法与前端 `testRunner.ts` 完全一致。

```python
async def execute_request(request: SendRequestInput, session: Session) -> ApiResponseData:
    # 1. 解析变量
    variables = request.variables or {}

    # 2. 执行 Pre Script（如果有）
    pre_result = {}
    if request.pre_script:
        pr = PreScriptRunner()
        pre_result = pr.run(request.pre_script, variables)
        # 应用 header/param/env/body 修改
        ...

    # 3. 发送 HTTP 请求（不变）
    result = await send_request(...)

    # 4. 执行 Tests 脚本（如果有）
    test_results = []
    if request.test_script:
        tr = TestRunner(result, variables)
        test_results = tr.run(request.test_script)

    # 5. 组装返回
    result.script_results = ScriptResults(
        pre_script_logs=pre_result.get("logs", []),
        test_results=[TestResultItem(**t) for t in test_results],
        env_changes=pre_result.get("env_changes", {}),
    )
    return result
```

## 五、Mock 模板变量

Mock 响应体变量替换在 `mock/handler` 中实现：

```python
# 在 mock_handler 中
import re
import uuid, time
from datetime import datetime

TEMPLATE_VARS = {
    "{{$timestamp}}": lambda: str(int(time.time())),
    "{{$uuid}}": lambda: str(uuid.uuid4()),
    "{{$datetime}}": lambda: datetime.now().isoformat(),
    "{{$randomInt}}": lambda: str(int(time.time() * 1000) % 1000000),
}

def replace_template_vars(text: str) -> str:
    for pattern, resolver in TEMPLATE_VARS.items():
        text = text.replace(pattern, resolver())
    return text
```

> ⚠️ 以下内容仅供爱因斯坦和产品经理参考，不下发给图灵。

### 8.1 关于 Pre Script 后端化的风险

当前前端 Pre Script 工作良好，迁移到后端可能引入新的 bug。**降级方案**：如果后端 Pre Script 实现复杂度超预期，保留前端执行，仅在 store 中将 `testScript` 通过 API 发给后端执行 Tests。

### 8.2 关于 Python `exec()` 安全

使用 `exec(script, {"pm": pm_obj, "__builtins__": {}})` 限制了内置函数访问（`__builtins__` 设为空字典），防止：
- 文件系统访问 (`open()`)
- 网络访问 (`__import__`)
- 进程操作 (`os.system()`)

但仍需在前端 UI 提示用户：Tests 脚本运行在受限沙箱中，不支持文件/网络操作。
