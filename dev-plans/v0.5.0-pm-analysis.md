## 二、v0.5.0 开发计划建议

### 产品经理判断：v0.5.0 的核心主题

**"完成脚本测试闭环，补齐响应体验，产出可分发版本"**

这是灵犀 API 从「能用」到「好用」的关键版本。理由：

1. **Tests/断言** 是 Postman 类工具的核心差异化能力——没有它，灵犀只是一个"发请求的工具"
2. **响应体验** 直接影响日常使用感受——HTML/图片/二进制是高频场景
3. **打包分发** 是项目可交付性的标志——让非开发者也能用

### v0.5.0 功能清单

#### 🔴 P0 — 必须交付（核心）

##### 1. Tests 脚本系统（阶段 5 完补）

**1.1 Tests 编辑器**
- 在请求编辑区新增 "Tests" 标签页，与 Params/Headers/Body/Auth 并列
- 包含 Pre Script 和 Tests 两个子视图
- Monaco Editor 编辑 JavaScript DSL
- 预置常用断言代码片段（一键插入）

**1.2 Tests 执行引擎**
- Post-response 执行：在 HTTP 响应返回后自动运行 Tests 脚本
- 执行上下文注入：`response`（状态码/headers/body/耗时）、`pm` 对象
- 安全沙箱：使用受限 JS 执行环境（如 PyMiniRacer 或 subprocess 隔离）

**1.3 断言 API**
```
pm.expect(value).toBe(expected)
pm.expect(value).toContain(substring)
pm.expect(value).toHaveStatus(code)
pm.expect(value).toHaveHeader(name, value)
pm.expect(value).toBeJson()
pm.expect(value).toBeLessThan(n)     // 耗时断言
pm.response.json()                   // 获取 JSON 响应
pm.environment.set(key, value)       // 延续现有
pm.environment.get(key)
pm.globals.set(key, value)
pm.globals.get(key)
```

**1.4 测试结果面板**
- 响应区新增 "测试结果" 标签页
- 显示：通过数/失败数/总计、每条断言的状态和详情
- 失败断言用红色高亮，附带期望值 vs 实际值对比

##### 2. 响应视图增强

**2.1 Body 预览视图**
- HTML 响应：内嵌 iframe 渲染预览（沙箱隔离）
- 图片响应：直接显示图片（检测 Content-Type）
- 二进制响应：下载按钮 + 十六进制预览（前 256 字节）

**2.2 Headers / Cookies 面板**
- 响应 Headers 表格展示（key/value，支持搜索）
- 响应 Cookies 面板（Name/Value/Domain/Path/Expires）
- 一键将响应 Cookie 添加到 Cookie Jar

**2.3 响应元信息**
- 响应区顶部信息栏细化：状态码(彩色)/耗时/大小/HTTP 版本
- 请求大小 vs 响应大小拆分显示

## 四、技术关注点（给爱因斯坦的提示）

1. **Tests 执行引擎选型**：前端受限 JS vs Python 后端执行 JavaScript DSL。推荐用 Python 后端执行，利用 `exec()` 做沙箱 + AST 安全校验，或考虑集成 `py_mini_racer`（V8 引擎的 Python 绑定）获得真正的 JS 语义
2. **Tests 执行时机**：在 http_client.py 的 send_request 完成后、返回前端之前执行
3. **HTML 预览安全**：iframe sandbox 属性 + CSP header，防止 XSS
4. **图片预览**：后端返回图片时判断 Content-Type，base64 编码后传给前端直接渲染
5. **数据模型变更**：需要新增 TestResult 模型、Capture 模型需要持久化字段
6. **不新增 feature 模块**：Tests 可作为 request_workbench 的子功能，不创建独立的 tests feature（减少路由复杂度）
